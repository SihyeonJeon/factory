## P0 — 바텀 시트 근본 수정

### HF2-1: 상세 보기 진입 시 상단 잘림
- 추억 간략히 보기 → 화면 확장 → 상세 보기 진입 시 바텀 시트 상단이 잘리는 현상
- 상세 보기에서는 콘텐츠가 status bar 아래부터 시작해야 함

### HF2-2: 바텀 시트 좌우 여백 제거
- 현재 바텀 시트가 좌우에 패딩이 있음
- 원래 화면 너비에 꽉 차야 함 (full-width)

### HF2-3: 추억 만들기(+) 버튼 위치
- 현재 위쪽에 동떨어져 있음
- 바텀 시트 핸들 바로 위에 위치해야 함
- 바텀 시트와 함께 움직여야 함

### HF2-6: 바텀 시트 스크롤/핸들링 여전히 부자연스러움
- 스크롤 + 핸들 드래그 동작이 아직 어색
- iOS 기본 시트처럼 자연스러운 제스처 필요
- 스크롤과 시트 드래그가 매끄럽게 전환되어야 함

### HF2-9: 바텀 시트 최대화 = 완전한 전체 화면
- 최대화했을 때 아직 상단에 핸들과 여백이 보임
- 최대화 시 핸들 숨김, 여백 제거, 완전한 하나의 화면이 되어야 함

## P1 — 탭 바 + 네비게이션 재설계

### HF2-7: 탭 구성 변경
- 현재: 지도 / 되감기 / 그룹 — 부자연스러움
- 리퀴드 글래스가 뒤의 내용을 가림
- **새 구성: 지도 / 캘린더 / 설정**
- "되감기"는 바텀 시트 메인 큐레이션에서 표시하면 됨, 탭 불필요
- **캘린더 페이지**:
  - 달력 뷰 (추억이 있는 날은 점으로 표시)
  - 날짜 선택 → 해당 날짜의 추억들 리스트
  - 연간 리캡 통합
  - 알림 설정 통합
  - 한 달 지출 비용 정리/요약
- **설정 페이지**:
  - 그룹 관리 (기존 GroupHubView 내용)
  - 프리미엄
  - 기타 설정
- 탭 바 스타일: 리퀴드 글래스 제거, 깔끔한 불투명 배경

## P2 — 콘텐츠 + 검색

### HF2-4: 사진 업로드 후 볼 수 없음
- 사진을 업로드해도 앱에서 표시되지 않음
- 업로드된 사진이 갤러리/상세 보기에서 보여야 함

### HF2-5: 클러스터 마커에 사진 표시
- 현재: 이모지 아이콘
- 원하는 것: 대표 사진 썸네일 (iOS Photos 앱처럼)
- 클러스터 내 첫 번째 추억의 사진을 대표 이미지로

### HF2-10: 검색 자동완성 크기 과대
- 검색 결과 리스트가 너무 크게 표시됨
- 더 컴팩트하게: 작은 글씨, 적은 패딩, 최대 높이 제한

### HF2-11: 장소명 검색 → 위치 정보 연동
- 현재 앱 내 데이터만 검색 가능
- **외부 지도 서비스(MapKit MKLocalSearch)로 장소 검색**
- 장소 검색 → 좌표 획득 → 추억 위치 지정에 활용
- 검색 바에서 장소 검색 시 지도상 위치까지 이동

## P3 — 테스트 인프라

### HF2-12: 더미 데이터로 테스트
- 현재 문제의 원인: 실제 데이터 없이 테스트
- SampleModels.swift에 풍부한 더미 데이터 생성:
  - 3개 그룹 (커플 1, 일반 2)
  - 5개 이벤트/데이트 (각 그룹에 분배)
  - 20개 추억 (다양한 장소, 날짜, 감정, 비용, 사진)
  - 날짜별 분포: 최근 1년 내 다양한 날짜
  - 위치 분포: 서울, 제주, 부산 등 다양한 좌표
