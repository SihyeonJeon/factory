### Disagree (substantive)
1. **"SKILLS.md ≥1 entry as Gate 5 blocker" is process theater.** Some rounds legitimately add nothing durable. Replace with: retro explicitly classifies whether new SKILLS entry is warranted; mandatory only when repeated/novel pattern found.
2. **"FILE_INDEX 경유 없이 bulk read 금지 + `ls <dir>` 금지" is impractical.** Debugging legitimately needs `rg --files`, `rg`, `git diff --name-only`, targeted `ls`. Ban undirected bulk-loading, log when opening unindexed files, but don't ban directory inspection outright.
3. **FILE_INDEX size contradicts itself** in proposal (table says "제한 없음", body says "≤250줄"). Pick one. Codex prefers 250 lines + topic-specific sub-indexes.

### Counter-proposals
- **Base `spec.md` immutable after lock creation.** Amendments live as `spec.amendment.N.md` + `supersedes` field in lock, or generate `spec.effective.md` by merging base + amendments. Direct edit to base during active round = hard stop.
- **Replace "mandatory disagreement" with "mandatory challenge section".** Normative/design decisions: ≥1 concrete objection, risk, or rejected alternative. Pure factual meetings: "No normative decision; verified facts only" + evidence pointers. Fake disagreement worse than none.
- **Split Gate 5** into `process-integrity-blockers` (missing lock mapping, contract mutation, unreviewed meeting, missing FILE_INDEX entry for new operator docs) and `process-advisories` (missing SKILLS update when retro deems none needed).
- **Reduce Codex load.** Don't make Codex own design + 1st verify + real-use eval. Split real-use eval: Codex writes rubric → Claude Code captures runtime evidence (screenshots, logs) → Codex reviews conclusions.

### Answers to 7 Open Questions
- **Q1 Gemini:** Keep as tier-3 escalation only (context overflow, deadlock arbitration, large-artifact summarization). Not in routine gates.
- **Q2 Meetings:** Markdown body + YAML frontmatter (round, stage, status, participants, decision_id, contract_hash). Append one JSONL event per decision to `process-log.jsonl`.
- **Q3 Disagreement scope:** Normative/design-affecting only. Factual meetings just cite evidence.
- **Q4 Screenshots:** Raw under `context_harness/reports/<round_id>/screenshots/`. Operator docs point via evaluation manifest; never dump raw into operator/.
- **Q5 Codex load:** As per counter-proposal above.
- **Q6 Deepsight first round:** SLICE. Capture/gap/contract → token sprint → nav sprint if needed → screen clusters → a11y sweep.
- **Q7 Sequence:** Minimal v5 bootstrap first (OPERATOR, FILE_INDEX, STAGE_CONTRACT, MEETING_PROTOCOL, lock schema, one dry-run meeting), **then** deepsight as first real v5 round. Don't perfect v5 before product, don't process deepsight under v4.

### Missing Pieces (a-g)
- **(a) Lint enforcement:** `context_harness/operator/lint_config.yaml` + `harness/check_operator_round.py`. Round lock stores lint config hash. Gate 5 fails closed on missing config / hash mismatch / checker failure. Enforced by the stage's **cross-validator**, not performer.
- **(b) Contract immutability:** Hashes alone insufficient. Protocol must declare base files never edited after lock; changes require amendment meeting + new amendment artifact; direct base diff during active round = hard stop.
- **(c) Operator-layer drift:** Operator audit script that compares OPERATOR/FILE_INDEX/STAGE_CONTRACT/MEETING_PROTOCOL/REGULATION/PROCESS_AUDIT/SESSION_RESUME for contradictions (role matrix mismatch, stale paths, line-limit violations, unindexed new files, divergent stage names). Runs at Gate 5 and every operator amendment.
- **(d) Non-convergent meetings:** After 2 written counter-rounds OR 24 hours without convergence → `status: escalated`. Summarize both positions, list reversible default, ask human for decision. No silent compromise, no endless loop.
- **(e) Parallel vs serial:** Parallel for independent evidence gathering and Gate 2 evaluators. Serial for contract-setting, approval, merge. Implementation parallel only with disjoint whitelists/worktrees. Phase 1 gap analysis parallel, Phase 2 contract update serial.
- **(f) Ownership:** Codex owns design/spec drafts, acceptance, eval rubrics, architecture review, `AGENTS.md`. Claude Code owns implementation briefs, Swift code, runtime execution, `.claude/CLAUDE.md`. `operator/` shared — requires meeting approval. Crossing zones → meeting + whitelist expansion + performer/cross-validator assignment before edits.
- **(g) Autonomy vs human approval:** Human required for product-contract changes, user-visible design direction, security/privacy policy, release/merge with unresolved blockers, non-convergent operator disputes. Human NOT required for sprint slicing inside approved product scope, implementation within locked whitelist, in-scope test fixes, process-log/report generation.

### Recommendation
**merge-with-changes.** Proposal is strong foundation but enforcement layer underspecified and several rules risk process theater (mandatory SKILLS, forced dissent on facts, overly strict read discipline). Add fail-closed lint/lock mechanics, immutable contract amendments, operator-doc drift checks, deadlock escalation, clearer Codex load split. Then v5 is ready for deepsight.

9. Autonomy inside locked contracts. Human required for product/security/architecture/irreversible/post-advisory deadlock.
10. Deepsight slicing: token → nav (if needed) → screen clusters → a11y sweep.
11. Sequence: minimal v5 bootstrap → deepsight round 1 under v5.
12. Enforcement: `operator/lint_config.yaml` + `harness/check_operator_round.py`. Lock stores hash. Fail-closed.
13. Drift audit: operator-layer drift script at Gate 5 and every operator amendment.
