## Challenge Section

### Objection
Codex's item 7 (promote missing-paths to blocker) is technically correct but I pushed for an allowlist — some operator docs legitimately reference future paths (e.g., `operator/indexes/`) or shorthand grep hints (`workspace/ios/.../UnfadingTheme.swift`). Promoting to hard blocker without allowlist would force us to create stub files for every forward reference. Codex's recommendation implicitly accepts allowlist. No objection recorded; this is alignment.

### Risk
`PurePosixPath.match` behavior with `**` varies slightly across Python versions. Python 3.13+ supports it properly. Fallback to fnmatch+manual `**` handling if behavior differs. Risk: mismatch at a future upgrade. Mitigation: unit-style assertion in checker self-test on first call.

### Rejected alternative
Implementing full STAGE_CONTRACT matrix parsing for role validation (REG §7 full claim). Rejected because: (a) STAGE_CONTRACT matrix is markdown table, fragile to parse; (b) this round's goal is drift resolution, not feature addition; (c) REGULATION narrowing is cheaper and more honest. If a future round genuinely needs role-matrix validation, open a peer meeting.

### Uncertainty deferred
`gate_evidence.json` schema — I'm defining a minimal schema inline but haven't specified rigid versioning. First real round will stress-test it. If operators diverge on format, open a follow-up meeting.
